Hey — I evaluated the `bemora` npm package (v3.6.0, unified API client, 94+ providers, single maintainer, repo: Demon-radio/Bemora.lol) for company-level production use. It works for hobby stuff but has real blockers. I want you to fork it internally and ship a `bemora-enterprise` v1.0 in one pass. Here's the full scope — do it all in one PR (or one coordinated set of PRs), not phased.

═══════════════════════════════════════════════════════════
PART A — FIX EXISTING BUGS (non-negotiable)
═══════════════════════════════════════════════════════════

1. CLI + MCP shebang bug: `src/cli.js` and `src/mcp-server/index.js` start with a blank line then `#!/usr/bin/env node` on line 2, which Node's ESM parser rejects. Move the shebang to line 1 in both files. Verify `npx bemora crypto bitcoin` works after the fix.

2. Add a real LICENSE file (MIT, matching package.json) at repo root. Add SECURITY.md with a vulnerability reporting policy. Add CHANGELOG.md starting at the fork version.

3. Add `.github/workflows/ci.yml` running the existing 13 unit tests + integration tests on Node 18/20/22, plus `npm audit` and a license-check step. No `.github` folder ships today — unclear if upstream even has CI.

4. Add axios `timeout: 8000` + `AbortController` signal to EVERY provider. Currently only ~10 of 94 providers set a timeout; a hung upstream hangs the whole service. The `signal` plumbing already exists in function signatures, just nobody passes it through.

5. Replace every `throw new Error(string)` with structured error classes:
   - `BemoraError` (base, has `provider`, `code`, `requestId`)
   - `BemoraProviderError` (extends base, adds `httpStatus`, `upstreamRequestId`)
   - `BemoraRateLimitError` (extends BemoraProviderError, adds `retryAfter`, `limit`, `remaining`)
   - `BemoraTimeoutError`, `BemoraAuthError`
   Update all ~30 provider files to throw these.

6. Centralize User-Agent into `src/core/headers.js`. Today there are 5 different UAs (`bemora/1.0`, `bemora-npm-library`, `bemora (https://...)`, etc.). Wikipedia 403'd because of this. One constant, exported, used everywhere.

7. Add request/response log redaction. The built-in logger logs full URLs; if a key is in the querystring (`appid=`, `apikey=`, `Authorization=`) it leaks. Add a logger interceptor that scrubs these before any `logger.info()`.

8. Fix the README repo links — badges point to `github.com/bemora/bemora` which doesn't exist. Point to the new internal fork.

9. Replace top-level `await import()` in `src/index.js` and `src/core/cache.js` with dynamic imports inside async functions so webpack/esbuild can consume the package without `target: 'es2022'`.

═══════════════════════════════════════════════════════════
PART C — ADD PLATFORM FEATURES (not providers, capabilities)
═══════════════════════════════════════════════════════════

1. Multi-tenant key management: `api.withTenant(tenantId, keys)` returns a scoped sub-instance. Per-customer keys for SaaS use cases. Backed by a tenant registry so the same provider can serve N tenants with different keys.

2. Key rotation: `api.keys.rotate('stripe', newKey)` — invalidates caches, swaps the live key atomically, emits a `keys:rotated` event.

3. Circuit breaker: `core/circuit.js`. After N consecutive failures from a provider, open the circuit for a cooldown window. `api.circuit.status('weather')` returns `{ state: 'closed'|'open'|'half-open', failures, lastFailure }`. Wire into the retry layer.

4. Cost tracking: emit `ai:response` events with `{ provider, model, inputTokens, outputTokens, costUsd }`. Add `api.costs.snapshot()` returning per-provider/per-tenant spend. Critical for AI products.

5. PII redaction: `core/pii.js`. Auto-scrub emails, phones, SSNs, card numbers from logs AND from data sent to LLMs (configurable). Apply via the middleware layer.

6. Request signing helpers: `signing/awsSigV4.js`, `signing/hmac.js`, `signing/cloudflare.js`. Today every caller writes their own.

7. Pagination helper: `api.helpers.paginate(fn, { cursor | offset | page })` — unified across cursor/offset/page strategies. Each provider exports its pagination type.

8. GraphQL client: `api.gql(endpoint, query, variables, { headers })`. Many modern APIs (Shopify, GitHub v4, Hasura) are GraphQL-only.

9. Multipart upload support: `api.helpers.upload(url, file, { fields, headers })`. Needed for S3 presigned POST, Cloudinary, etc.

10. Schema enforcement: every provider exports a zod schema for its response. Validate on the way out. Reject malformed data with `BemoraProviderError` instead of crashing callers. zod is already a dep, just barely used today.

11. Fallback chains beyond AI: extend `core/fallback.js` to non-AI providers. Ship with: weather → OpenWeatherMap → Open-Meteo → WeatherAPI → NWS; news → NewsAPI → RSS aggregation; currency → ExchangeRate-API → Frankfurter → ECB. One dead upstream no longer breaks the app.

12. Redis cache adapter: `core/cache-redis.js`. The library already exposes `cache.setAdapter(adapter)` — just ship the Redis implementation so multi-instance deploys share cache.

13. Webhook receiver router: `api.webhooks.route('stripe', req)` — parse, verify, dispatch to handler. Today bemora only does outbound; inbound webhooks are manual.

14. Per-provider axios instances: `core/http.js` exports `httpClient({ timeout, retries, headers })` factory. Every provider uses it instead of raw `axios.get`. Centralizes timeout/retry/header/interceptor config.
